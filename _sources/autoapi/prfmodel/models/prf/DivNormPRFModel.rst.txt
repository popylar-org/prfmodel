prfmodel.models.prf.DivNormPRFModel
===================================

.. py:class:: prfmodel.models.prf.DivNormPRFModel(activation_prf_model: prfmodel.models.base.BasePopulationResponse, normalization_prf_model: prfmodel.models.base.BasePopulationResponse, shared_params: list[str] | None = None, encoding_model: prfmodel.models.base.BaseStimulusEncoder | type[prfmodel.models.base.BaseStimulusEncoder] = PRFStimulusEncoder, impulse_model: prfmodel.impulse.base.BaseImpulse | type[prfmodel.impulse.base.BaseImpulse] | None = DerivativeTwoGammaImpulse, scaling_model: prfmodel.scaling.base.BaseScaling | type[prfmodel.scaling.base.BaseScaling] | None = DivNormAmplitude, regressors_model: prfmodel.regressors.base.BaseRegressors | list[prfmodel.regressors.base.BaseRegressors] | None = None)



   Divisive normalization population receptive field model.

   Receives two independent pRF responses (activation and normalization) and combines them
   via the divisive normalization formula.
   Parameters that should be shared between both responses (e.g. the pRF centre
   ``mu_x``, ``mu_y``) are listed in ``shared_params`` and appear once in
   :attr:`parameter_names` without a suffix. All remaining pRF parameters are suffixed
   with ``_activation`` or ``_normalization`` respectively.

   :param activation_prf_model: pRF response model (activation).
   :type activation_prf_model: BaseResponse
   :param normalization_prf_model: pRF response model (normalization).
   :type normalization_prf_model: BaseResponse
   :param shared_params: Names of pRF parameters that are shared between the two responses.  Each name
                         must appear in *both* ``activation_prf_model.parameter_names`` and
                         ``normalization_prf_model.parameter_names``.
   :type shared_params: list of str, default=["mu_x", "mu_y"]
   :param encoding_model: An stimulus encoding model class or instance. Model classes will be instantiated during initialization.
                          The default creates a :class:`~prfmodel.models.prf.PRFStimulusEncoder` instance.
   :type encoding_model: BaseStimulusEncoder or type, default=PRFStimulusEncoder
   :param impulse_model: An impulse model class or instance. Model classes will be instantiated during
                         initialization. The default creates a :class:`~prfmodel.impulse.DerivativeTwoGammaImpulse`
                         instance with default values.
   :type impulse_model: BaseImpulse or type or None, default=DerivativeTwoGammaImpulse, optional
   :param scaling_model: A scaling model class or instance. Model classes will be instantiated during initialization.
                         The default creates a :class:`~prfmodel.scaling.DivNormAmplitude` instance.
   :type scaling_model: BaseScaling or type or None, default=DivNormAmplitude, optional
   :param regressors_model: A regressor model instance, a list of regressor model instances, or `None`. When a list is
                            provided, it is wrapped in a :class:`~prfmodel.regressors.RegressorsList` and its contributions
                            are summed. The regressor contribution is added after the scaling model.
   :type regressors_model: BaseRegressors or list of BaseRegressors or RegressorsList or None, default=None, optional

   .. rubric:: Notes

   The predicted response is:

   .. math::

       p_{\text{DN}} = \frac{(a R_1 \cdot S + b)}{(c R_2 \cdot S + d)} - \frac{b}{d}

   Where `R_1` and `R_2` are the activation and normalization pRF responses, `S` is the stimulus.
   The :math:`-b/d` term ensures a zero response in the absence of a stimulus.


   .. py:method:: __call__(stimulus: prfmodel.stimuli._prf.PRFStimulus, parameters: pandas.DataFrame, regressors: pandas.DataFrame | None = None, dtype: str | None = None) -> prfmodel.typing.Tensor

      Predict the DN model response to a stimulus.

      :param stimulus: Population receptive field stimulus object.
      :type stimulus: PRFStimulus
      :param parameters: Dataframe with columns containing different model parameters and rows containing parameter values
                         for different units.
      :type parameters: pandas.DataFrame
      :param regressors: Regressor design data. Required when the canonical model has a regressors model configured.
                         A single data frame with shape `(num_frames, num_regressors)` whose columns cover the names
                         required by every configured regressor model. Extra columns are ignored.
      :type regressors: pandas.DataFrame, optional
      :param dtype: The dtype of the prediction result. If `None` (the default), uses the dtype from
                    :func:`prfmodel.utils.get_dtype`.
      :type dtype: str, optional

      :returns: The predicted model response with shape `(num_units, num_frames)` and dtype `dtype`.
      :rtype: Tensor



   .. py:method:: predict_responses(stimulus: prfmodel.stimuli._prf.PRFStimulus, parameters: pandas.DataFrame, dtype: str | None = None) -> prfmodel.typing.Tensor

      Predict each of the two responses before applying the DN formula.

      :returns: Stacked predictions of shape (num_voxels, 2, num_frames), where axis 1 index 0
                is the activation response and index 1 is the normalization response.
      :rtype: Tensor



   .. py:property:: parameter_names
      :type: list[str]


      Names of parameters used by the model.

      Shared parameters appear once (no suffix). Response-specific parameters are suffixed
      with ``_activation`` or ``_normalization``.
