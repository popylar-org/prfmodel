prfmodel.models.cf.canonical.CanonicalCFModel
=============================================

.. py:class:: prfmodel.models.cf.canonical.CanonicalCFModel(cf_model: prfmodel.models.base.BasePopulationResponse, encoding_model: prfmodel.models.base.BaseStimulusEncoder | type[prfmodel.models.base.BaseStimulusEncoder] = CFStimulusEncoder, scaling_model: prfmodel.scaling.base.BaseScaling | type[prfmodel.scaling.base.BaseScaling] | None = BaselineAmplitude)



   Canonical connective field model.

   This class combines a connective field and scaling model response.

   :param cf_model: A connective field response model instance.
   :type cf_model: BasePopulationResponse
   :param encoding_model: An encoding model class or instance. Model classes will be instantiated during initialization. The
                          default creates a :class:`~prfmodel.models.encoding.PRFStimulusEncoder` instance.
   :type encoding_model: BaseStimulusEncoder or type, default=PRFStimulusEncoder
   :param scaling_model: A scaling model class or instance. Model classes will be instantiated during initialization.
                         The default creates a :class:`~prfmodel.models.scaling.BaselineAmplitude` instance.
   :type scaling_model: BaseScaling or type or None, default=BaselineAmplitude, optional

   .. rubric:: Notes

   The simple composite model follows three steps:

   1. The connective field response model makes a prediction for the stimulus distance matrix.
   2. The connective field response is encoded with the source response.
   3. The scaling model modifies the encoded response.

   In contrast to pRF models (e.g., :class:`~prfmodel.models.CanonicalPRFModel`), connective field models do not
   require an impulse model because it already contained in the signal of the source response.
