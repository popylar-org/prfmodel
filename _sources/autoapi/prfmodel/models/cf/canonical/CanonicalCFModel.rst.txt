prfmodel.models.cf.canonical.CanonicalCFModel
=============================================

.. py:class:: prfmodel.models.cf.canonical.CanonicalCFModel(cf_model: prfmodel.models.base.BasePopulationResponse, encoding_model: prfmodel.models.base.BaseStimulusEncoder | type[prfmodel.models.base.BaseStimulusEncoder] = CFStimulusEncoder, scaling_model: prfmodel.scaling.base.BaseScaling | type[prfmodel.scaling.base.BaseScaling] | None = BaselineAmplitude, regressors_model: prfmodel.regressors.base.BaseRegressors | list[prfmodel.regressors.base.BaseRegressors] | None = None)



   Canonical connective field model.

   This class combines a connective field and scaling model response.

   :param cf_model: A connective field response model instance.
   :type cf_model: BasePopulationResponse
   :param encoding_model: An stimulus encoding model class or instance. Model classes will be instantiated during initialization.
                          The default creates a :class:`~prfmodel.models.cf.CFStimulusEncoder` instance.
   :type encoding_model: BaseStimulusEncoder or type, default=CFStimulusEncoder
   :param scaling_model: A scaling model class or instance. Model classes will be instantiated during initialization.
                         The default creates a :class:`~prfmodel.scaling.BaselineAmplitude` instance.
   :type scaling_model: BaseScaling or type or None, default=BaselineAmplitude, optional
   :param regressors_model: A regressor model instance, a list of regressor model instances, or `None`. When a list is
                            provided, it is wrapped in a :class:`~prfmodel.regressors.RegressorsList` and its contributions
                            are summed. The regressor contribution is added after the scaling model.
   :type regressors_model: BaseRegressors or list of BaseRegressors or RegressorsList or None, default=None, optional

   .. rubric:: Notes

   The canonical model follows the following steps:

   1. The connective field response model makes a prediction for the stimulus distance matrix.
   2. The connective field response is encoded with the source response.
   3. The scaling model modifies the encoded response.
   4. The regressors model (optional) adds a linear combination of fixed regressors to the scaled response.

   In contrast to pRF models (e.g., :class:`~prfmodel.models.CanonicalPRFModel`), connective field models do not
   require an impulse model because it already contained in the signal of the source response.

      :type regressors: pandas.DataFrame, optional

      :type dtype: str, optional

      :returns: The predicted model response with shape `(num_units, num_frames)` and dtype `dtype`.
      :rtype: Tensor
